# TypePulse — Build Plan

> Generated by Claude Opus (claude-opus-4-6) · March 2026

---

## Product Summary

TypePulse is a Monkeytype-inspired typing practice app that replaces random words with real knowledge. Users pick a category — **Latest AI News** or **Current Affairs** — and type through live article excerpts pulled from free news APIs. No account, no database, no install. Just a URL, a keyboard, and something worth knowing. The target user is a developer, student, or curious professional who wants to sharpen their typing speed while actually absorbing information.

---

## Tech Stack

| Layer | Technology |
|---|---|
| Framework | Next.js 16 (App Router) |
| Language | TypeScript 5 |
| Styling | Tailwind CSS 4 |
| Components | shadcn/ui + Radix UI (minimal — mostly custom) |
| Fonts | Geist Mono (typing area), Geist Sans (chrome) |
| Animations | Framer Motion 12 |
| News sources | NewsAPI.org (free tier) + GNews.io (fallback) |
| State | `useState` / `useReducer` — no external store |
| Sharing | URL search params (`?wpm=94&acc=97&cat=ai-news`) |
| Deployment | Vercel (free tier) |
| Analytics | Vercel Analytics |

**No database. No auth. No server-side persistence of any kind.**

API keys are kept server-side via Next.js Route Handlers — never exposed to the browser.

## Phase 1: News Fetching Pipeline

**Goal**: Reliable server-side news fetch that returns clean, typeable article text.

**Duration**: 1–2 days

**Deliverables**:
- [x] `lib/categories.ts` — two category configs:
  ```ts
  { slug: 'ai-news', label: 'AI News', query: 'artificial intelligence OR LLM OR OpenAI', sources: 'techcrunch,wired,the-verge' }
  { slug: 'current-affairs', label: 'Current Affairs', category: 'general' }
  ```
- [x] `lib/news.ts`:
  - `fetchFromNewsAPI(category)` — calls NewsAPI, parses response
  - `fetchFromGNews(category)` — fallback if NewsAPI rate-limited
  - `normaliseToTypingText(article)` — extracts title + first 1–2 sentences, strips: em dashes → `-`, smart quotes → `'`/`"`, HTML entities, `[+N chars]` artifacts, parenthetical URLs
  - Returns `{ id, title, body, source, url, publishedAt }` where `body` ≤ 280 chars
- [x] `app/api/news/route.ts` — wires `fetchArticles()`, returns 5 articles, cached 10 min via `next: { revalidate: 600 }`
- [ ] Manual QA: log 20 articles, check every character is standard ASCII and text ends on a word boundary

**Key files**:
- `lib/news.ts`
- `lib/categories.ts`
- `app/api/news/route.ts`

**Success criteria**: `GET /api/news?category=ai-news` returns 5 clean articles. No API key visible in browser network tab. All body text is clean ASCII, no truncated words.

## Phase 2: Typing Engine

**Goal**: Core typing game loop — character state machine, WPM, accuracy, timer.

**Duration**: 2 days

**Deliverables**:
- [x] `lib/typing.ts`:
  - `type CharState = 'pending' | 'correct' | 'incorrect'`
  - `calcWPM(correctChars: number, elapsedMs: number): number` — standard formula: `(correctChars / 5) / (elapsedMs / 60000)`
  - `calcAccuracy(correct: number, total: number): number`
- [x] `hooks/useTypingEngine.ts`:
  - Input: `targetText: string`
  - Tracks: `charStates: CharState[]`, `typedCount`, `startTime`, `isStarted`, `isFinished`
  - Starts timer on first keypress
  - Handles backspace — reverts last char to `'pending'`
  - Finishes when `typedCount === targetText.length`
  - Exposes: `{ charStates, wpm, accuracy, isStarted, isFinished, reset }`
  - WPM recalculated every 300ms via `setInterval` while typing
- [x] Global `keydown` listener on `document` — no `<input>` element, raw keyboard capture
- [x] `Esc` key resets session

**Key files**:
- `lib/typing.ts`
- `hooks/useTypingEngine.ts`

**Success criteria**: Hardcode a 100-char string, type it in the browser. Backspace works. WPM updates live. Finishing the text sets `isFinished = true`. Colors correct (green/red/gray).

## Open Questions

1. **Article length**: Title only (~60 chars, ~30s session) vs title + 1–2 sentences (~250 chars, ~90s)? Recommend starting with title + 1 sentence for a satisfying session length.
2. **NewsAPI rate limits**: Free tier = 100 req/day. With 10-min caching = ~1,400 sessions/day before hitting limits. Add GNews fallback on day one, not after.
3. **Tab behaviour**: Pre-fetch 5 articles on page load and cycle locally (fast, stale) vs fetch fresh on each Tab press (slow, always fresh)? Recommend pre-fetch batch, refresh batch when exhausted.
4. **Third category**: Tech / Science / Finance — when? Only after launch validation shows which category gets more plays.
5. **Result persistence**: Should results be stored in `localStorage` for a personal history view? Not in v1 — adds complexity, defer until there's user demand.

## Risks

| Risk | Likelihood | Mitigation |
|---|---|---|
| NewsAPI 100 req/day limit hit at launch | High | 10-min cache + GNews fallback live on day one; add TheNewsAPI.com as third source |
| Article text contains untyable characters | High | Build sanitiser in Phase 1, manually QA 50 articles before launch |
| Monkeytype adds a "news mode" | Low | Ship in 2 weeks; brand and content curation are the moat |
| NewsAPI bans scraping / changes ToS | Low | Two fallback sources already planned; all fetch server-side so easy to swap |
